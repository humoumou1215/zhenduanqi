# 任务提示词模板

本文档定义了常用任务的标准化提示词模板，确保 Agent 能够正确遵循项目规则。

---

## 开发任务模板

### 模板 1：解决 Issue（标准流程）

```
解决 issue #{编号}：

1. 遵循项目规则（.trae/rules/project_rules.md）
2. 从 main 检出 feature 分支（如已有未提交改动，先 git stash）
3. 使用 TDD 技能开发（RED → GREEN → REFACTOR）
4. PRD 随代码同步更新（同一个 commit）
5. 推送分支并创建 PR
6. 审查 PR，决策合并或拒绝
```

**使用场景**：标准的 issue 解决流程，适用于新功能开发和 Bug 修复。

---

### 模板 2：解决 Issue（快速模式）

```
解决 issue #{编号}（快速模式）：

1. 遵循项目规则
2. 从 main 检出 feature 分支
3. 实现代码并更新测试
4. PRD 随代码同步更新
5. 推送分支并创建 PR
```

**使用场景**：简单的改动，不需要严格的 TDD 流程。

---

### 模板 3：查看并选择 Issue

```
查看还有哪些 issue，按优先级和时间顺序选择 1 个 issue：

1. 使用 gh issue list 查看所有 open 的 issue
2. 按优先级排序（P0 > P1 > P2）
3. 同优先级按创建时间排序（早的优先）
4. 使用"解决 Issue（标准流程）"模板解决选中的 issue
```

**使用场景**：需要 Agent 自动选择 issue 并解决。

---

## 审查任务模板

### 模板 4：审查 PR

```
审查 PR #{编号}：

1. 检查 PR 是否遵循项目规则：
   - [ ] PRD 同步规则（PRD-1 至 PRD-5）
   - [ ] Git 工作流规则（GIT-1 至 GIT-15）
   - [ ] 测试覆盖是否充分
   - [ ] 代码质量是否符合标准
2. 运行测试确保通过
3. 决策：合并或拒绝
4. 如果合并，使用 squash merge 并删除分支
```

**使用场景**：审查已有的 PR。

---

## 文档任务模板

### 模板 5：更新 PRD

```
更新 PRD：

1. 识别改动类型（新功能/Bug 修复/重构/删除）
2. 确定需要更新的 PRD 章节（参考 PRD 更新章节检查清单）
3. 在同一个 commit 中提交 PRD 更新
4. 提交信息格式：docs(prd): {描述} (#{issue-number})
```

**使用场景**：单独的 PRD 更新任务。

---

## 技能使用模板

### 模板 6：需求探讨

```
探讨 {需求描述}：

1. 使用 grill-me 技能逐项确认需求
2. 确认后使用 to-prd 技能输出 PRD
3. 使用 to-issues 技能拆分为 issue
```

**使用场景**：新功能规划、技术方案选型。

---

## 快捷命令模板

### 模板 7：QPRD

```
qprd
```

触发 PRD 同步检查流程。

### 模板 8：QCHECK

```
qcheck
```

触发代码变更检查流程。

### 模板 9：QISSUE

```
qissue
```

触发规范 issue 创建流程。

---

## 注意事项

1. **所有开发任务都必须从检查分支状态开始**
2. **禁止直接在 main 分支上提交代码**
3. **PRD 更新必须与代码改动在同一个 commit 中**
4. **每个 PR 必须关联一个 issue**
5. **TDD 模式适用于核心业务逻辑开发**
